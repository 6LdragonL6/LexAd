# LexAd v0.5.1 发布说明

> 历史版本文档：本文仅记录 v0.5.1 当时的实现，不代表当前审查逻辑。请查看 [v0.7.0 文档中心](../../README.md) 和 [v0.7.0 发布说明](../../releases/v0.7.0-release-notes.md)。

日期：2026-07-08

## 概述

v0.5.1 在 v0.5.0 的稳定工作流基础上新增明暗双模式，并把前端样式逐步收敛到主题 token。随后补入可靠性与演示体验整合：修复误导性审查完成态、保留审查输入快照、完善有条件通过流转，并为文件解析加入资源边界。

## 新增能力

### 明暗双模式

- 支持浅色、深色、跟随系统三种主题偏好。
- 首次访问默认跟随系统 `prefers-color-scheme`。
- 用户手动选择后保存在本地浏览器。
- 主题切换即时生效，不需要刷新页面。

### 全局主题入口

- 顶部栏新增主题切换控件。
- 移动端和桌面端共用同一套主题状态。
- 版本显示改为读取前端常量，避免 UI 中继续硬编码版本号。

## 结构与样式优化

- 新增 `frontend/src/composables/useTheme.ts` 管理主题偏好、系统主题监听和根节点属性。
- 新增 `frontend/src/constants/app.ts` 维护应用名称和版本号。
- 在 `frontend/src/styles/main.css` 中定义语义化 CSS 变量：
  - 页面与卡片背景
  - 主/次/弱文字
  - 边框
  - 品牌色
  - 成功、警告、危险、信息状态色
- 更新 UnoCSS 共享快捷类，让 `card`、`input`、`btn-*`、`page-heading` 等基础 UI 元素默认支持主题。
- 为现有页面中常见的浅色 utility class 增加暗色模式兜底，减少一次性大规模重写风险。

## 文档与版本

- 前端包版本更新为 `0.5.1`。
- 后端项目与运行时健康检查版本均为 `0.5.1`。
- README、开发指南、技术参考更新到 v0.5.1。
- v0.5.0 发布说明保留为历史记录。

## 可靠性与工作流整合

- 法律审查失败不再被标记为已完成，也不会展示误导性的 0 分结论；中断超过 15 分钟的任务会在应用启动时恢复为可重试失败状态。
- 每个物料版本首次提交审查时保存不可变输入快照，版本历史可返回当时正文、行业、平台和优先级。
- 法务“有条件通过”成为独立终态，必须填写条件，并在市场与法务工作台中明确显示。
- 创建与更新物料共用字段限制；文件提取移出 API 事件循环，图片、PDF 和提取文本设有上限。
- 后端打包元数据已包含实际运行的模型、认证、OCR 与 Office/PDF 解析依赖。

## 已知限制

- 本版本没有拆分 `AdminKnowledgePage.vue` 等大型页面组件。
- 浏览器原生 `confirm` / `alert` 仍未替换为统一弹窗。
- 暗色模式通过主题 token 和兼容覆盖实现，后续新增页面仍应优先使用语义化 shortcut。
- 暂未加入视觉回归测试。
