# LexAd v0.7.0 发布说明

> 本文为历史版本说明。当前版本请参阅 [v0.7.1 发布说明](../../releases/v0.7.1-release-notes.md)。

发布日期：2026-07-15

v0.7.0 重构了法律、平台和舆情审查的核心判断方式。系统不再把规则命中、关键词或数字片段直接作为用户可见结论，而是将其作为内部候选交给 AI 结合完整语境和资料库裁决，再验证原文证据与引用依据。

## 1. 版本重点

### 资料库约束下的 AI 裁决

- 法规、行业资料、平台规则和已核验案例作为模型优先参考。
- 硬规则与关键词只召回候选，不直接生成违规结论或扣分。
- 法律、平台和舆情按不同风险逻辑分别审查，避免互相替代。
- 模型引用只能指向本轮输入中真实存在的资料标识。

### 风险与核验事项分离

- 已确认风险必须具有完整原文证据、判断理由和有效资料依据。
- 销量、数据来源、检测、认证、资质等真实性问题进入独立核验清单。
- “需要证明”不再被直接表达为“已经违规”。

### 安全降级

- AI 不可用、资料库为空、平台没有生效规则或引用验证失败时，系统标记人工复核。
- 候选规则不会在失败时回退成确定性违规。
- 舆情审查不再使用本地触发词或相似度结果生成自动风险结论。

## 2. 审查引擎变化

### 法律与平台

审查流水线调整为：

```text
候选召回 → 分领域 AI 裁决 → 原文与依据验证 → 风险 / 核验 / 人工复核
```

- 第一阶段只记录规则候选数量，不向用户暴露原始命中。
- 法律 AI 同时接收完整文案、行业、资料库资源、候选规则和相似案例。
- 每个所选平台使用当前生效规则版本独立裁决。
- 重复问题按原文证据和风险类型合并。
- 法律合规分只由已确认法律和平台风险扣减。

### 舆情

- 本地触发词与案例只作为模型输入候选。
- AI 必须逐字引用语义完整的物料原文。
- 相似事件只有被 AI 明确选择且标识有效时才展示。
- AI 失败时返回不确定并要求人工复核，不显示本地词条命中理由。

## 3. 输出与界面

- 结果页区分“已确认风险”“资料核验事项”和“待人工复核”。
- 法律与平台问题展示完整原文、中文风险等级、判断理由、资料依据和建议。
- 舆情结果不再展示原始匹配词、分词或内部相似度。
- 用户可见风险标题统一为中文，不再使用 `high`、`medium`、`low` 作为小标题。
- 历史抽屉和法务详情采用相同的可信输出规范。

## 4. 接口模型变化

审查结果新增或扩展：

- `BasisReference`：资料标识、标题和版本；
- `VerificationItem`：待核验声明、类型、原因、材料和依据；
- `MatchedRule.evidence_quote`、`reasoning`、`risk_level_label`、`confidence`、`basis_refs`；
- 分层结果的 `candidate_count`、`verification_items`、`requires_manual_review`；
- 总结果的 `review_status`、`verification_items`、`requires_manual_review`。

依赖旧 `matched_text`、关键词命中或本地舆情融合字段的前端与外部消费者需要适配。字段可能仍为兼容性保留，但不应作为用户可见结论来源。

## 5. 评分与状态

法律合规分从 100 分开始，只按已确认风险扣减：

| 等级 | 扣分 |
| --- | ---: |
| 高风险 | 30 |
| 中风险 | 15 |
| 低风险 | 5 |

新增业务状态：

- `no_clear_risk`：未发现明确风险；
- `needs_verification`：存在资料核验事项；
- `confirmed_risk`：存在已确认风险；
- `manual_review`：依据或服务不足，需要人工复核。

舆情继续使用独立风险等级与风险分，不计入法律合规分。

## 6. 升级注意事项

1. 前后端版本均升级为 `0.7.0`。
2. 本版本没有新增数据库迁移，但升级前仍应备份并执行 `alembic upgrade head`。
3. 完整自动审查需要可用的 DeepSeek API 配置；未配置时会出现人工复核状态。
4. 平台审查需要管理员维护当前生效且具有结构化规则的版本。
5. 法规与行业资料目录必须可读，否则法律结果会提示人工复核。
6. 外部消费者应优先读取 `evidence_quote`、`basis_refs`、`verification_items` 和 `review_status`。

## 7. 验证结果

发布前验证：

- 后端：`python -m pytest app/tests -q`，116 项测试通过；
- 前端：`npm run build`，类型检查和生产构建通过；
- 文档与代码差异：`git diff --check` 通过；
- 真实模型连通场景已验证法律核验分流、舆情语境判断和平台资料缺失降级。

真实模型验证只用于确认链路行为，不构成准确率或法律效果承诺。

## 8. 已知限制

- AI 和资料库都可能存在遗漏，引用校验不能消除全部误判。
- 平台规则覆盖取决于管理员维护的生效版本。
- 文案以外的产品事实、授权、报告和实际投放方式仍需人工核验。
- 当前后台任务为进程内任务，不适合直接承担高并发、跨实例的生产级队列需求。
- LexAd 是企业广告合规辅助工具，不替代律师意见或监管机关判断。
