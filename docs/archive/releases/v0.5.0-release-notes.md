# LexAd v0.5.0 发布说明

> 历史版本文档：本文仅记录 v0.5.0 当时的实现，不代表当前审查逻辑。请查看 [v0.7.0 文档中心](../../README.md) 和 [v0.7.0 发布说明](v0.7.0-release-notes.md)。

日期：2026-07-08

## 概述

v0.5.0 是一个"修复优先"的稳定发布。在 v0.4.2 的基础上修复了三项测试人员报告的实测 bug，补齐了退回物料循环提交流程，接入了团队提供的真实舆情案例数据，并统一了前端展示语言。

## Bug 修复

### 平台规则别名匹配

**问题：** 提交页保存中文平台名（如"抖音""小红书"），但管理员平台规则使用内部标识（如 `douyin`）。L4 匹配只做精确字符串比较，导致平台规则集存在时仍被判定为"暂无生效规则"。

**修复：** 新增平台别名归一化映射表，支持以下别名互认：

| 展示名称 | 内部标识候选 |
|---|---|
| 抖音 | douyin, douyin_ec, 巨量引擎 |
| 小红书 | xiaohongshu, red, xhs |
| 微信/视频号 | wechat, weixin, shipinhao |
| 微博 | weibo |
| 京东 | jd, jingdong |
| 淘宝 | taobao, tmall |
| 拼多多 | pdd, pinduoduo |

匹配逻辑：用户选择和规则集名称分别归一为别名集合，任一交集即视为同平台。

### 市场部退回信息可见

**问题：** 市场部查看被退回物料时，看不到法务退回原因和内部备注，无法按法务意见修改。

**修复：**
- 市场部工作台（首页和"我的提交"页）直接展示退回原因、法务备注。
- 提供"修改并重新提交"和"归档"按钮。
- 导航菜单对市场用户显示"我的提交"入口。

### L1 "最"类绝对化表达检测

**问题：** 知识源中有"最（含'最'字的绝对化用语）"概括性规则，但运行时只保留有限枚举词，"最耐用"漏检；"国家级品质"虽命中"国家级"但报告没有解释命中逻辑。

**修复：**
- L1 拆为精确词 + 模式规则两套检测。
- 新增"最"类模式检测：匹配"最" + 1~4 个性能/品质描述词（如"最耐用""最安全""最省电"）。
- 边界控制：不把"最近""最终""最低价"（已被精确词覆盖）当成广告主张误判。
- 每条命中附带业务语言解释：命中词、风险类型、法律依据、修改建议。

## 新功能

### 退回物料循环提交

- `MaterialStatus` 新增 `archived`（已归档）。
- 编辑退回物料时如果 `raw_text`、`industry` 或 `platforms` 变化，版本号自动递增。
- 重新发起 AI 审查时使用新版本号。
- 已有审查记录的物料只能归档，不能物理删除；仅无审查记录的草稿可删除。
- 版本列表返回每次提交的 AI 结果、法务决定、退回原因和备注。
- 前端显示"第 1 次提交 / 第 2 次提交"，不暴露数据库版本号。

### 舆情数据接入

- 接入团队提供的 34 条真实舆情案例（`sentiment_cases.json`）。
- 接入 10 个维度的触发词库（`trigger_words.json`），用于舆情引擎快速路径扫描。
- 导入脚本基于 `external_id` 幂等：已发布事件跳过，草稿事件更新，不静默覆盖历史。
- 案例结构字段（行业、平台、风险维度、触发短语、严重级别）直接写入，无需模型二次处理。
- 触发词命中影响舆情风险等级和建议文案。

### 报告人性化

技术表达替换为业务语言：
- "未命中硬规则" → "本层未发现明确违禁词或硬性风险点"
- "未命中平台规则" → "本次未发现与该平台生效规则直接冲突的表达"
- "资料库待补充" → "当前舆情案例库不足，暂不能给出案例化判断"
- 平台状态区分"暂无规则集"和"已有规则集但暂无生效版本"

### 前端展示规范化

- 所有业务页面统一使用中文平台名，不出现拼音式内部标识。
- 内部标识仅供技术详情折叠区查看。
- 版本快照主展示格式为"抖音 / 2026-07"，不是裸 UUID。

## 已知限制

- 实时舆情抓取和平台规则自动爬取不在本版本范围内。
- L1 边界控制白名单为代码清单维护，暂无管理界面。
- "最"类模式检测对 1~2 个非描述词中文的匹配较保守。
- 舆情风险不计入法律合规分。
- 实时 PostgreSQL 迁移验证可能因网络限制无法在沙箱中完成。
