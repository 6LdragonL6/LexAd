# LexAd v0.6.1 发布说明

> 历史版本文档：本文仅记录 v0.6.1 当时的实现，不代表当前审查逻辑。请查看 [v0.7.0 文档中心](../../README.md) 和 [v0.7.0 发布说明](v0.7.0-release-notes.md)。

发布日期：2026-07-13

v0.6.1 聚焦于审查结果的可用性、审核闭环和窗口适配。

## 体验与布局

- 应用导航改为完整侧栏、图标侧栏和抽屉三级模式；在中间宽度优先保留桌面侧栏。
- 页面容器、工具栏和多栏页面支持连续重排，减少侧栏遮挡和控件越界。
- 版本号移动至侧栏或抽屉底部。
- 工作台按角色显示统计：市场端显示待处理、需修改、已通过；法务端显示待审核、已通过、已退回。

## 审查资料与报告

- 首次初始化会幂等导入内置 L3/L4 知识基线；管理员已有启用规则不会被覆盖。
- 新增拼多多基线和 `拼多多 / pdd / pinduoduo` 别名接入。
- 平台规则与舆情案例增加确定性关键词和中文短语相似性混合检索；模型不可用时仍保留可解释结果。
- 审查报告突出实际命中内容，将无命中模块压缩，并清晰区分资料不可用和执行失败。

## 审核历史与提醒

- 市场结果页和法务详情页可从右侧抽屉查看历史提交、审查报告和法务结论。
- 工作台黄色合规提示改为基于结构化资料池轮换，避免连续重复。

## 验证

- 后端测试覆盖知识基线、拼多多别名、历史版本和既有审查流程。
- 前端生产构建通过。
