# LexAd v0.6.0 发布说明

> 历史版本文档：本文仅记录 v0.6.0 当时的实现，不代表当前审查逻辑。请查看 [v0.7.0 文档中心](../../README.md) 和 [v0.7.0 发布说明](v0.7.0-release-notes.md)。

发布日期：2026-07-12

v0.6.0 聚焦界面体验、品牌资料沉淀和本地开发可靠性。它保留原有市场、法务、管理员业务闭环，不改变法律合规与舆情风险双轴审查的基本口径。

## 新功能：品牌库

- 新增品牌模型、迁移、Schema、服务和 API。
- 市场部可在提交物料时选择或创建品牌，并记忆最近使用的品牌。
- 品牌档案聚合品牌关联物料、审核状态、通过率、平均提交版本和最近活动。
- 支持品牌编辑、归档以及明确的列表、空状态、错误和重试反馈。
- 内置幂等演示品牌数据，重复初始化不会创建重复记录。

本版本不会在本地 SQLite 与 Neon 之间自动同步品牌数据。正式环境切换到 Neon 前，应先迁移表结构，再通过单独流程导入需要保留的数据。

## 新 UI 与响应式布局

- 登录页与登录后的应用壳层严格分离。
- 桌面端采用深色固定侧栏；紧凑窗口和手机端采用顶部栏与导航抽屉。
- 登录页在桌面和平板保持左右结构，在手机宽度切换为上下结构。
- 页面支持从 320px 手机宽度到宽屏桌面的连续重排。
- 首页、提交页、结果页、法务详情、品牌档案和资料中心统一使用流式容器。
- 页面只产生纵向主滚动；宽表格和标签栏在自身容器中横向滚动。

## 主题与可访问性

- 扩充页面、外壳、卡片、抬升面板、内嵌区域和悬停状态的语义化颜色。
- 深色模式增加背景与面板层级，避免所有区域呈现近似颜色。
- 桌面与移动端均提供明确的退出登录入口。
- 关键输入和按钮保持适合触控的最小高度，窄窗口允许工具栏和按钮组换行。

## 数据库模式

- `local` 是日常开发默认模式，严格使用本地 SQLite。
- `neon` 必须显式选择并提供 PostgreSQL 连接字符串。
- 应用与 Alembic 共用数据库 URL 解析逻辑，避免运行和迁移指向不同数据库。
- 生产环境禁止静默回退到 SQLite。

## 启动流程

- Windows 一键启动支持显式数据库模式。
- 本地模式在启动服务前执行 Alembic 迁移和幂等种子数据。
- Neon 模式执行只读连接与迁移版本检查，不在日常启动中自动修改云数据库。
- 启动错误区分依赖、迁移、端口、后端健康检查和前端启动失败。

## 验证

- 前端执行 TypeScript 检查和 Vite 生产构建。
- 后端执行回归测试和数据库配置测试。
- 临时 SQLite 从空库执行到最新 Alembic revision，并重复运行种子数据验证幂等性。
- 响应式验收覆盖 320、360、390、768、1024、1279、1280 和 1440px 宽度。

## 已知边界

- 本地 SQLite 与 Neon 之间没有实时或双向同步。
- 品牌红线、确认流程、版本审计和高级统计属于后续版本。
- 真实手机仍需额外检查虚拟键盘、安全区域和动态浏览器地址栏。
