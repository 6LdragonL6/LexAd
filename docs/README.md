## 当前版本的关键认知

- 硬规则与关键词只召回候选，不直接产生违规结论。
- AI 优先依据系统提供的法规、行业资料、平台规则和已核验案例。
- 已确认风险必须通过完整原文和资料引用验证。
- 数据、认证、资质和来源等问题进入资料核验事项。
- AI 或资料不可用时转人工复核，不回退到本地关键词结论。
- 舆情安全分与法规合规分分别计分、分别解释，均为 0–100 且越高越好；无可靠自动结论时不伪造数值。
- 法律与舆情审核必须基于同一提交快照，法务在同一主报告中完成双轴复核。
- 品牌可维护多个标准行业，物料产生的新行业由管理员确认后沉淀。

## 维护约定

- [审查方法论](architecture/review-methodology.md)是审查行为的唯一事实说明。
- README 负责项目价值和快速入口，不复制全部技术细节。
- 技术参考记录当前实现；发布说明只记录版本变化。
- 当前指南文件名不包含版本号，后续直接在原路径维护。
- 历史文件不机械替换版本号，只增加历史提示和当前入口。
- 新增、移动或删除文档时必须更新本索引并检查相对链接。
- 版本、测试数量、效率与准确性表述必须能够复现或明确标注为评测方法。
