# LexAd v0.7.1 发布说明

发布日期：2026-07-15

v0.7.1 是一次版本一致性、品牌知识沉淀和法务协作体验更新。v0.7.0 的资料库约束、AI 语境裁决、引用验证和风险分流原则保持不变。

## 1. 版本重点

### 审核版本一致性

- 工作台与队列使用首次提交标题作为稳定识别名。
- 每次审核返回其不可变提交快照，详情和历史记录不再读取当前工作副本代替。
- 法律、平台和舆情引擎均读取同一快照。
- 重新提交通过单一原子接口创建新版本、快照和审核任务。

### 品牌与行业

- 重新提交保持首次品牌，不允许清空或更换。
- 一个品牌可关联多个标准行业。
- 选择品牌后，品牌常用行业在提交页优先展示，但不会自动选择。
- 物料使用的新行业形成管理员候选，可接受、忽略或恢复。
- 迁移会把既有品牌行业和历史物料行业转成正式关联或待确认候选。

### 0.7.1 补充修复

- 修复提交页创建品牌失败后名称立即消失的问题；创建成功后才清空并选中，失败时保留原文字、错误和重试入口。
- 品牌档案增加完整“现有记忆印象”，确定性归纳样本量、近期法务倾向、修改轮次、高频风险和常见建议。
- 审核、提交、结果和品牌档案共用同一品牌记忆数据，分别采用紧凑与完整展示。
- Windows 一键启动默认关闭后端热重载，健康检查改用 `127.0.0.1` 并等待最多 60 秒。
- 后端提前退出时启动脚本显示退出码；健康超时时显示最后一次连接错误。

### 法务双轴审核

- 法务审核台以中栏为主要内容区，本次文案与审查引擎居中展示。
- 审查引擎同时展示法律/平台报告和完整舆情报告。
- 舆情仍在处理时不能提交法务决定。
- 舆情失败或不可用时，法务完成人工复核确认后可继续决定。
- 历史版本同步展示当次提交、法律报告、舆情报告和法务结论。

### 明暗模式与响应式

- 超宽屏采用辅助左栏、主中栏和操作右栏。
- 中等屏幕优先保留主报告和操作区域，移动端按阅读顺序单栏展示。
- 新增状态色、标签、卡片和提示均适配明暗主题，保持一致的风险语义与对比度。

## 2. 数据与接口变化

- `materials` 增加稳定标题 `display_name`。
- 新增 `brand_industries` 与 `brand_industry_suggestions`。
- Review 输出增加绑定的 `submission` 快照。
- 新增原子重提接口和标准行业目录接口。
- 品牌输出增加 `industries`，管理员可管理行业候选。
- 品牌档案输出增加确定性 `memory_impression`；不新增数据库表或迁移。
- 法务决定请求增加人工舆情复核确认字段。

迁移头为 `b8d4e1f7a203`。升级前应备份生产数据库，随后执行：

```bash
cd backend
alembic upgrade head
```

## 3. 兼容说明

- 旧审核缺少提交快照时以只读兼容提示展示，不使用当前文案伪装历史版本。
- 品牌原自由文本行业在迁移后不再作为主要业务数据源。
- 旧的通用物料更新接口不再承担已退回物料的重新提交职责。
- v0.7.0 的审查评分和 AI 裁决逻辑没有改变。

## 4. 验证结果

- 后端完整回归：122 项测试通过。
- Alembic 空 SQLite 数据库可从初始版本升级至 `b8d4e1f7a203 (head)`。
- 前端 TypeScript 检查通过。
- Windows 启动脚本 PowerShell 语法与稳定模式断言通过。
- 前端生产构建通过（Vite 完成 151 个模块转换）。
