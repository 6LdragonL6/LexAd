# LexAd v0.7.2 发布说明

发布日期：2026-07-15

## 主要变化

- 提交物料后展示“物料已接收、法规审核、舆情审核、汇总审查结果”四个真实阶段；
- 法规合规分与舆情安全分统一为 0–100、越高越好，无可靠舆情结论时不伪造数值；
- 法务选择通过且 AI 已确认风险时增加二次确认，提交失败保留输入并允许重试；
- 工作台和法务审核台进入详情后按可信来源返回，所有加载、处理、成功和失败状态均有返回按钮；
- 保持浏览器原生后退，法务决定成功后避免返回可重复提交的旧表单；
- 新增进度、风险确认、评分和返回逻辑的前端单元测试，并完善明暗主题与减少动画支持。

## 数据库与 API 变更

本版本包含破坏性字段迁移：

- `reviews.ai_risk_score` 重命名为 `reviews.legal_compliance_score`；
- 新增可空列 `reviews.public_opinion_safety_score`；
- 可靠历史舆情数据按新方向一次性回填，人工复核、失败或不可用记录保持空分；
- 历史舆情 JSON 中的数值评分被移除，证据、解释、置信度和建议保留；
- 新 API 只返回 `legal_compliance_score`、`public_opinion_safety_score` 和结构化 `stages`，不保留旧评分别名。

前后端必须作为同一个 v0.7.2 发布单元升级。旧前端不能长期与新后端混用。

## 升级步骤

1. 确认目标数据库模式和地址；
2. 创建可恢复数据库备份；
3. 部署同版本前后端；
4. 在后端目录执行 `alembic upgrade head`；
5. 检查健康接口版本为 `0.7.2`；
6. 抽查无风险、带风险和人工复核三类历史及新增审核；
7. 验证工作台返回、法务台返回和浏览器后退。

如果需要回退应用，必须先停止新版本写入，再将代码与 Alembic 迁移一起降级，不能只回退代码。
