# LexAd v0.4.1 迭代报告

日期：2026-07-05

## 迭代目标

本版本优先解决三个使用端问题：

1. 提交后长期停留在“审查进行中”，页面没有退出和失败恢复能力。
2. 法规数据库无法直接浏览本地 L1-L5 知识库。
3. 法务用户能够看到物料提交入口。

同时修复检查中发现的高风险安全问题，并保持现有 Vue/FastAPI 架构和天蓝色视觉体系。

## 已完成修改

### 后台审查

- Review 增加 `task_status`、`error_message`、`started_at`、`completed_at`。
- `POST /reviews/ai-review` 创建任务后以 HTTP 202 立即返回。
- 后台任务使用独立 SQLAlchemy Session 执行审查。
- 成功后物料进入 `pending_legal`；失败后保存错误并恢复为可重试状态。
- 防止同一物料并发创建多个运行中的审查任务。
- 结果页每 2 秒轮询，包含处理中、长耗时提示、网络重试、失败、重新审查和返回工作台。

### 法规数据库

- 按 L1-L5 五层展示知识库。
- L2/L3/L4/L5 按现有子目录继续分组。
- 点击条目在站内查看 TXT 正文。
- 未增加搜索功能。
- 正文接口限制在 `knowledge/` 目录内并拒绝路径穿越。

### 角色和业务权限

- 法务用户不显示提交物料导航。
- `/submit` 仅允许市场用户和管理员访问。
- 后端提交、文件预览和触发 AI 审查同样限制为市场用户或管理员。
- 市场用户只能读取自己的物料和审查。
- 法务用户只能读取进入法务流程的物料。
- 法务决定不能重复覆盖。
- 首页不再把 Material ID 错当作 Review ID。

### 导航体验

- 原顶部横向功能栏替换为左上角大尺寸菜单按钮。
- 点击后从左侧展开 300px 抽屉，移动端宽度不超过视口的 86%。
- 功能入口使用约 56px 高的纵向菜单项，提升鼠标和触屏点选体验。
- 用户名称、角色和退出登录移动到抽屉底部，降低顶栏信息密度。
- 当前页面保持天蓝色高亮。
- 支持菜单选择后自动关闭、遮罩关闭、关闭按钮和 Esc 关闭。
- 市场、法务和管理员继续使用同一角色过滤规则。

### 安全和稳定性

- 原始广告文本先进行 HTML 转义，再插入受控高亮，修复存储型 XSS。
- 临时上传文件使用服务端 UUID 文件名，修复客户端文件名目录穿越。
- 用户编辑后的预览文本作为最终正文，避免文件提取结果重复拼接。
- DeepSeek 配置、客户端创建、请求和 JSON 解析错误统一进入任务失败状态。
- AI 调用错误不再伪装成违规规则参与扣分。
- 移除实际无法可靠解析的旧二进制 `.doc` 格式，保留 `.docx`。
- CI 后端测试失败会正确阻断流水线。
- Docker Compose 增加知识库和 ChromaDB 挂载。
- Render 启动命令在启动 API 前执行 `alembic upgrade head`。
- 修正前端生产 API 环境变量为 `VITE_API_BASE_URL`。

## 数据库升级

升级前建议先备份 Neon PostgreSQL，然后执行：


## 本地服务关闭提醒

- 前端和后端运行期间，应在各自终端按 `Ctrl + C`，等待服务退出后再关闭窗口。
- 如果直接关闭终端后端口仍被占用，使用 `netstat -ano | findstr :5173` 和 `netstat -ano | findstr :8000` 查询 PID。
- 确认 PID 后使用 `taskkill /PID <PID> /T /F` 结束对应进程树。
- 不要批量结束全部 Node 或 Python 进程，以免影响其他项目。

## 单人项目版本管理建议

推荐继续采用语义化版本：

```text
主版本.次版本.修订版本
0.4.1
```

- 修订版本：漏洞修复、小型 UI 修复、兼容性修改，例如 `0.4.1 → 0.4.2`。
- 次版本：新增完整功能且兼容现有数据，例如 `0.4.x → 0.5.0`。
- 主版本：产生不兼容 API、数据库或产品定位变化，例如 `0.x → 1.0.0`。

每次发布建议执行：

```bash
git checkout -b release/v0.4.1
git status
cd backend
python -m pytest app/tests -q
cd ../frontend
npm run build
cd ..
git add <逐项确认的文件>
git commit -m "release: v0.4.1"
git tag -a v0.4.1 -m "LexAd v0.4.1"
git push origin release/v0.4.1
git push origin v0.4.1
```

单人维护仍建议使用短分支：

- `main`：始终保持可部署。
- `feature/<name>`：新功能。
- `fix/<name>`：缺陷修复。
- `release/vX.Y.Z`：发布前集中验证。

不要把 API Key、数据库密码、`.env`、本地 ChromaDB 或构建产物提交到 Git。每个版本至少保留：

1. 更新后的 README。
2. 数据库迁移文件。
3. 迭代报告。
4. 测试结果。
5. Git tag。
