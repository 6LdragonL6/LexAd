# LexAd v0.4.2 迭代报告

日期：2026-07-07

## 迭代目标

v0.4.2 用于解决 0.5 更新前必须闭环的三个问题：

1. 提交物料入口需要降低实操复杂度，让市场人员先完成核心提交，再按需展开高级项。
2. 平台规则需要管理员可更新、可启用、可回滚，并在审核结果中保留版本快照。
3. 除法律合规风险外，系统需要开始识别潜在舆情风险，并在舆情资料库为空时给出明确提示。

本版本仍属于 0.5.0 前置版本。真实舆情 JSON 数据补入后，才进入 v0.5.0。

## 已完成能力

### 管理员资料中心

- 新增管理员“资料中心”页面。
- 支持舆情案例列表、关键词筛选、状态管理和详情查看。
- 支持自然语言录入舆情文本、事件经过、后果等材料。
- 支持调用统一模型服务，将自然语言材料整理为结构化舆情案例。
- 支持发布、归档、恢复和草稿删除。
- 支持导入记录和操作日志展示。

### 舆情 JSON 导入

- 新增舆情案例 JSON 模板接口。
- 支持粘贴 JSON 后预览导入结果。
- 支持校验必填字段、重复 `external_id`、有效条数和错误条数。
- 支持确认导入、跳过重复项或更新重复项。
- 支持导入后自动调用统一模型服务整理结构化版本。
- 导入功能面向普通管理员，不要求其接触数据库或后端文件。

### 平台规则版本管理

- 新增平台规则集和平台规则版本模型。
- 支持管理员创建平台规则集。
- 支持创建规则版本，记录来源、正文、生效时间和结构化规则。
- 支持版本差异摘要，展示新增、修改、删除数量。
- 支持启用新版本和回滚旧版本。
- 审核结果记录当次使用的平台规则版本 ID。

### 审核引擎

- L4 平台审查接入管理员维护的平台规则。
- 审核记录新增法律模块状态和舆情模块状态。
- 舆情风险作为独立风险轴，不覆盖法律合规风险。
- 舆情引擎支持确定性触发、相似案例聚合和统一模型服务解释。
- 当舆情资料库为空时，结果返回“资料库待补充”，不生成虚假的低风险结论。
- 统一模型服务/API 接口继续复用原法律风险审查调用通道，内部按任务类型区分提示词和解析逻辑。

### 前端体验

- 提交物料页改为核心字段优先：广告内容、行业、平台。
- 行业选择支持多选；后端继续使用 `industry: string` 兼容存储，多个行业以 `、` 拼接，审查引擎会拆分后匹配多行业规则。
- 高级项默认收起，降低首次提交成本。
- 结果页新增法律风险和舆情风险双轴展示。
- 结果页显示舆情命中主题、相似案例、模型说明和资料库状态。
- 结果页显示平台规则版本快照。
- 顶部菜单显示 v0.4.2。

### 逻辑完整性修正

- 舆情风险聚合不再因为“仅行业相同”就直接继承历史案例的高严重度。
- 历史严重度只在命中强度达到阈值时上调风险，降低行业泛匹配导致的误报。

## 管理员操作建议

### 舆情资料

1. 优先使用单条自然语言录入。
2. 有批量资料时使用 JSON 导入。
3. AI 整理后必须人工复核。
4. 只有确认无误的案例才发布。
5. 过期或不再适用的案例使用归档，不直接删除。

### 平台规则

1. 每个平台建立一个规则集。
2. 每次平台规则更新建立一个新版本。
3. 填写来源链接和生效时间。
4. 启用前查看差异摘要。
5. 发现新版本有误时回滚旧版本。

## v0.5.0 前置条件

进入 v0.5.0 前，需要补充真实舆情 JSON 文件，并至少完成：

1. 导入一批真实案例。
2. 人工复核结构化结果。
3. 用典型广告素材回归测试舆情风险提示。
4. 根据误报和漏报调整触发词、案例标签和模型提示词。
5. 确认平台规则的来源和更新责任人。
