# LexAd v0.6.2 发布说明

发布日期：2026-07-14

v0.6.2 是一次面向管理员能力与运行稳定性的小版本更新。

## 管理员系统管理

- 新增“系统管理”页面，仅管理员可访问。
- 管理员可保存、替换、测试和清除单一 DeepSeek API Key。
- DeepSeek Base URL 固定为 `https://api.deepseek.com`，模型固定为 `deepseek-v4-flash`。
- API Key 加密保存，页面、接口、日志和审计记录均不返回明文。
- 生产环境仍使用默认 `SECRET_KEY` 时拒绝保存 API Key。

## 回收站

- 物料及关联审核记录、品牌、舆情案例和平台规则可由管理员移入回收站。
- 数据保留 15 天，期间可恢复或手动永久删除；到期后由系统自动清理。
- 删除品牌不会删除历史物料；永久清理品牌时解除历史关联。
- 导入记录与审计日志不可删除并永久保留。

## 模型调用稳定性

- 语义合规审查、舆情案例结构化和舆情风险解释统一接入 DeepSeek 网关。
- 输入统一规范化并限制长度，输出使用 JSON 模式和 Pydantic 结构校验。
- 对超时、限流和上游服务错误执行最多两次重试；认证和格式错误不重试。
- 模型模块失败时保留确定性规则结果，并明确展示失败状态。

## 登录修复

- 错误账户或密码不再触发整页重载。
- 登录成功直接返回用户信息，消除令牌已保存但用户加载失败的半登录状态。
- 增加请求超时、重复提交保护和网络错误提示。
- 会话过期处理只作用于携带会话令牌的受保护请求。

## 验证

- 后端回归测试覆盖加密配置、固定模型网关、回收站关系、认证响应和既有审核流程。
- Alembic 从空数据库升级到 v0.6.2 通过，并验证 v0.6.2 到上一版本的回退步骤。
- 前端 TypeScript 类型检查和 Vite 生产构建通过。
