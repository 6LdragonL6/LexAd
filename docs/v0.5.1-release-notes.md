# LexAd v0.5.1 发布说明

日期：2026-07-08

## 概述

v0.5.1 是一个主题与结构打磨版本。它在 v0.5.0 的稳定工作流基础上新增明暗双模式，并把前端样式逐步收敛到主题 token，降低后续页面维护成本。

本版本不改变广告审查引擎、法务流转规则或后端 API 行为。

## 新增能力

### 明暗双模式

- 支持浅色、深色、跟随系统三种主题偏好。
- 首次访问默认跟随系统 `prefers-color-scheme`。
- 用户手动选择后保存在本地浏览器。
- 主题切换即时生效，不需要刷新页面。

### 全局主题入口

- 顶部栏新增主题切换控件。
- 移动端和桌面端共用同一套主题状态。
- 版本显示改为读取前端常量，避免 UI 中继续硬编码版本号。

## 结构与样式优化

- 新增 `frontend/src/composables/useTheme.ts` 管理主题偏好、系统主题监听和根节点属性。
- 新增 `frontend/src/constants/app.ts` 维护应用名称和版本号。
- 在 `frontend/src/styles/main.css` 中定义语义化 CSS 变量：
  - 页面与卡片背景
  - 主/次/弱文字
  - 边框
  - 品牌色
  - 成功、警告、危险、信息状态色
- 更新 UnoCSS 共享快捷类，让 `card`、`input`、`btn-*`、`page-heading` 等基础 UI 元素默认支持主题。
- 为现有页面中常见的浅色 utility class 增加暗色模式兜底，减少一次性大规模重写风险。

## 文档与版本

- 前端包版本更新为 `0.5.1`。
- 后端项目版本更新为 `0.5.1`，但后端业务逻辑未改动。
- README、开发指南、技术参考更新到 v0.5.1。
- v0.5.0 发布说明保留为历史记录。

## 验证命令

```powershell
cd frontend
npm.cmd run build
```

如果后续在本版本追加后端行为改动，再运行：

```powershell
cd backend
python -m pytest app/tests/ -q
```

## 已知限制

- 本版本没有拆分 `AdminKnowledgePage.vue` 等大型页面组件。
- 浏览器原生 `confirm` / `alert` 仍未替换为统一弹窗。
- 暗色模式通过主题 token 和兼容覆盖实现，后续新增页面仍应优先使用语义化 shortcut。
- 暂未加入视觉回归测试。
