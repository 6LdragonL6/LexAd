# LexAd 测试物料批量验证 — 设计文档

## 目标

用 `物料1-65.xlsx`（65 条标注广告案例）通过 API 端到端验证 LexAd 审查引擎，确认合规/违规二分类 + 风险等级的判定与期望结果基本一致，无明显偏差。

## 测试物料

- 路径：`D:\LDragonl\桌面\物料1-65.xlsx`
- Sheet：Sheet1，66 行（1 行表头 + 65 条数据）
- 列：物料编号、行业、子行业、物料形式、合规状态、违规类型一级/二级、严重程度、风险等级、广告内容、模型审核结果、涉及法律法规、来源说明
- T001-T055：违规案例（覆盖医疗健康、金融、食品、教育培训、化妆洗护、房地产、汽车、电子数码、旅游酒店、酒类、直播带货、通用）
- T056-T065：合规案例

## 架构

```
┌──────────┐     HTTP/JSON      ┌──────────────┐     SQLite     ┌──────────┐
│ 验证脚本  │ ──── POST/GET ──── │ FastAPI 后端  │ ───────────── │ lexad.db │
│ validate │                    │ (需预先启动)   │               │          │
│ .py      │                    │              │               │          │
└──────────┘                    └──────────────┘               └──────────┘
```

- 脚本不修改任何引擎或业务代码，纯 HTTP 客户端
- 后端负责完整的请求处理流水线：认证 → 物料创建 → AI 审查 → 结果返回

## 对比逻辑

### 合规/违规判定

引擎输出映射到二分类的规则：

| 条件 | 判定 |
|------|------|
| `ai_risk_score < 80` | 违规 |
| `ai_result` 中 L1/L2/L3 任一层的 `matched_rules` 非空 | 违规 |
| 以上都不满足 | 合规 |

### 风险等级映射

| ai_risk_score | 风险等级 |
|---------------|----------|
| < 50 | 高 |
| 50 — 79 | 中 |
| ≥ 80 | 低 |

### 偏差定义

| 级别 | 条件 |
|------|------|
| 严重偏差 | 合规/违规二分类判断错误 |
| 一般偏差 | 二分类正确，但风险等级差一档或以上 |
| 通过 | 二分类 + 风险等级完全一致 |

## 脚本设计

### 路径

`backend/scripts/validate_materials.py`

### 依赖

- `openpyxl` — 读取 xlsx
- `requests` — HTTP 调用

### 执行流程

1. **登录**：`POST /api/v1/auth/login`，使用预置 `market01 / test1234`，获取 JWT token
2. **逐条测试**（65 次循环）：
   - `POST /api/v1/materials/submit` — 创建物料
     - `name` = 物料编号（如 "T001"）
     - `industry` = xlsx 中的行业
     - `raw_text` = xlsx 中的广告内容
     - `material_type` = xlsx 中的物料形式
   - `POST /api/v1/reviews/ai-review` — 触发 AI 审查，body: `{"material_id": "<id>"}`
   - `GET /api/v1/reviews/by-material/{id}` — 获取审查结果（`ai_risk_score` + `ai_result`）
   - 对比期望 vs 实际，记录偏差
   - 收集 `material_id` 到清理列表
3. **输出报告**：
   - 终端打印汇总：总准确率、二分类准确率、风险等级准确率、偏差案例清单
   - 输出 CSV 文件 `validation_report.csv` 到脚本同级目录，包含每条物料的详细对比
4. **清理**（`finally` 块保证）：
   - 遍历收集的 `material_id` 列表
   - 通过数据库 session 删除 Material 记录（级联删除关联 Review）
   - 即使用户 Ctrl+C 也会尝试清理已创建的记录

### 错误处理

- 单条 API 调用失败（超时/500）→ 记录错误，跳过该条，继续下一条
- 登录失败 → 立即退出，报告原因
- 清理失败 → 打印警告，提示手动清理

## 成功标准

- 二分类准确率 ≥ 85%（55 条违规 + 10 条合规中，分类错误不超过 10 条）
- 风险等级准确率 ≥ 75%（允许相邻档位的偏差，严重跨档视为不通过）

## 不在范围内

- 不验证"违规类型一级/二级"的匹配（仅验证二分类 + 风险等级）
- 不验证"严重程度"字段
- 不修改引擎代码或 prompt（验证完成后如需调优，另开任务）
- 不做 CI 集成
