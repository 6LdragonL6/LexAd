# LexAd v0.6 发布收尾设计

## 背景

v0.6 品牌库与新界面完成后，联调暴露出几组相互关联的问题：登录页视觉和控件间距不稳定；固定侧栏、三栏页面和窄窗口使用不同断点；深色主题缺少清晰的表面层级；品牌档案没有可靠的加载状态；本地 SQLite 与 Neon 的选择不够明确；一键启动难以区分迁移、连接和服务启动失败。

本文合并原先分散的启动可靠性、UI/数据库修复和响应式布局设计。品牌库的核心数据与接口设计仍见 [v0.6 品牌库主设计](2026-07-11-lexad-v0.6-ui-brand-library-design.md)。

## 目标

1. 登录页与登录后应用壳层完全隔离，并保持参考 UI 的品牌视觉。
2. 从 320px 手机宽度到宽屏桌面连续适配，无页面级横向滚动、遮挡或控件重叠。
3. 桌面固定侧栏、紧凑端抽屉和复杂多栏页面使用一致的断点。
4. 深色主题形成页面、卡片、抬升面板和内嵌区域的视觉层级。
5. 品牌列表和档案加载具有明确状态、错误分类与重试能力。
6. 日常开发明确使用本地 SQLite；只有显式选择时才连接 Neon。
7. 一键启动在服务启动前完成必要预检，并输出可诊断的失败原因。

## 非目标

- 不实现 SQLite 与 Neon 的实时或双向同步。
- 不引入大型前端组件框架。
- 不复制参考 HTML 中登录页与应用内容并排、侧栏比例固定等交互缺陷。
- 不在普通本地启动中自动迁移或写入 Neon。
- 不在本版本实现品牌红线、品牌版本或复杂缓存。

## 应用壳层与响应式规则

| 宽度 | 导航 | 登录页 | 复杂内容 |
| --- | --- | --- | --- |
| `>=1280px` | 固定深色侧栏和桌面顶栏 | 左品牌区 + 右表单 | 允许三栏或双栏 |
| `768–1279px` | 顶部栏 + 抽屉 | 保持左右结构并压缩留白 | 最多双栏，辅助信息下移 |
| `320–767px` | 顶部栏 + 抽屉 | 上下结构 | 单列、自动换行 |

`AppShell` 是唯一负责桌面侧栏占位的组件。固定侧栏出现时主内容同时增加对应偏移；低于断点时侧栏和偏移必须同时移除。页面使用流式容器、`min-width: 0` 和响应式内边距。表格可以在自己的包装容器内横向滚动，但不能扩大文档宽度。

页面高度不设上限，内容通过纵向滚动访问。只有导航抽屉、遮罩和确有必要的局部区域使用固定定位或独立滚动。

## 登录页

登录页继续由独立 `AuthLayout` 承载，不渲染 `AppShell`。桌面与平板保持左侧渐变品牌区和右侧表单面板；手机端将品牌区压缩为页头式区域并隐藏大装饰和能力列表。

表单使用自然内容高度。标题、输入框、错误消息、登录按钮和测试账号区按稳定的纵向间距排列；控件最小触控高度为 44px。矮窗口允许整页向下滚动，不通过垂直居中挤压内容。

## 深色主题

深色主题使用语义化表面层级：

| 用途 | 色值 |
| --- | --- |
| 页面底色 | `#070D19` |
| 应用外壳 | `#0B1220` |
| 普通卡片 | `#111B2E` |
| 抬升面板 | `#17243A` |
| 内嵌区域 | `#0D1627` |
| 悬停区域 | `#1D2D47` |
| 边框 | `#2A3A55` |

通用卡片、输入框、状态徽章和布局组件优先使用语义令牌；仅为尚未迁移的 UnoCSS 工具类保留兼容映射。

## 退出登录

桌面顶栏和移动抽屉都提供明确的退出入口。两个入口调用相同的用户状态清理逻辑，删除本地令牌并使用路由替换进入 `/login`，避免后退直接回到受保护页面。

## 品牌档案状态

品牌列表状态为 `loading`、`ready`、`empty`、`error`。网络不可达、403、服务端错误和未知错误提供不同提示；401 继续由统一认证拦截器处理。档案加载失败不清空已经成功加载的品牌列表，并提供独立重试入口。

归档成功后，品牌从只返回 active 状态的当前列表中移除，同时清空选择和档案区域。品牌 API 使用具体响应模型约束返回结构。

## 数据库模式

数据库模式由 `DATABASE_MODE` 显式选择：

- `local`：只接受 SQLite URL，默认 `sqlite:///./lexad.db`。
- `neon`：必须提供 PostgreSQL URL，缺失或类型错误时拒绝启动。

应用和 Alembic 调用同一 URL 解析函数。生产环境禁止使用 local 模式。启动日志只显示脱敏后的数据库类型，不输出完整连接字符串。

本地测试期间产生的数据只存在于 SQLite。正式切换 Neon 时，先人工确认并执行迁移，再通过独立的一次性流程导入需要保留的数据。

## 一键启动

Windows 启动脚本支持显式数据库模式，并按以下顺序执行：

1. 检查 Python、虚拟环境、Node、npm、端口和必要文件。
2. local 模式执行 `alembic upgrade head` 和幂等种子数据。
3. neon 模式只读检查连接和当前 revision，不自动迁移。
4. 启动后端并等待健康检查成功。
5. 启动前端并等待端口可用。
6. 保存本次启动的进程信息，供关闭脚本精确停止。

PowerShell 参数必须位于脚本首部；脚本文本兼容 Windows PowerShell 5.1 编码；子进程命令延迟展开环境变量；Alembic revision 使用明确的正则匹配结果，而不是依赖共享 `$Matches` 状态。

## 验收

- 前端生产构建通过。
- 后端回归测试、数据库配置测试和品牌 API 测试通过。
- 空临时 SQLite 能迁移到最新 revision，重复种子不产生重复数据。
- 响应式检查覆盖 320、360、390、768、1024、1279、1280 和 1440px。
- 亮色与深色下均无页面级横向滚动、侧栏遮挡和控件重叠。
- Windows 完整验证 `start-dev.bat → 登录 → 品牌列表 → stop-dev.bat`。
