# LexAd v0.4.1 稳定性、安全性与角色体验设计

日期：2026-07-05  
版本：v0.4.1  
状态：待用户确认

## 1. 目标

本次迭代优先解决当前阻塞使用的问题，并在不替换现有技术栈、不大规模重构业务模型的前提下修复高风险漏洞：

1. 将 AI 审查改为后台执行，结果页通过状态轮询展示进度，避免页面无限卡住。
2. 法规数据库按 L1-L5 分类展示本地知识库，并支持站内查看正文，不增加搜索功能。
3. 法务用户不再看到或访问物料提交界面。
4. 修复越权访问、存储型 XSS、上传路径穿越和 AI 异常状态残留。
5. 保持现有天蓝色视觉体系和所有页面的交互完整性。
6. 更新 README、迁移说明和本次迭代报告，不执行 Git 提交或推送。

## 2. 范围

### 2.1 本次包含

- FastAPI 后台审查任务。
- 审查任务状态持久化和数据库迁移。
- 前端审查状态轮询、失败反馈、返回和重试操作。
- L1-L5 知识库分类、分组和正文查看。
- 前后端角色权限和资源归属校验。
- 安全修复及针对性自动化测试。
- v0.4.1 版本号、README 和迭代报告。

### 2.2 本次不包含

- Celery、RQ、Redis 等独立任务基础设施。
- 法规全文搜索、向量搜索或高级筛选。
- L3 证明材料引擎和 L4 平台适配引擎的完整重写。
- 用户注册、组织管理和复杂权限配置后台。
- 大规模 UI 重设计。

## 3. 方案选择

采用 FastAPI 后台任务、数据库任务状态和前端轮询。

该方案保持当前单体部署结构，不新增外部服务。与同步等待相比，它能够立即返回审查 ID 并让页面恢复控制；与 Celery/Redis 相比，它更适合当前单人维护和部署规模。局限是进程重启时正在执行的内存任务会中断，因此任务失败状态、超时恢复和后续可重试能力必须落到数据库。

## 4. 审查流程设计

### 4.1 数据模型

在 `reviews` 表增加：

- `task_status`：`processing`、`completed`、`failed`。
- `error_message`：保存经过截断和清理的用户可读错误。
- `started_at`：任务实际开始时间。
- `completed_at`：任务完成或失败时间。

新增 Alembic 迁移，不改动已有主键、外键和历史审查结果。

### 4.2 提交流程

1. 市场用户提交物料。
2. 前端请求创建 AI 审查。
3. 后端检查物料归属、状态和是否已有运行中的任务。
4. 后端创建 `processing` 状态的 Review，提交数据库后立即返回 Review。
5. FastAPI 后台任务使用独立数据库会话执行审查引擎。
6. 成功时写入审查结果、评分和 `completed`，物料转为 `pending_legal`。
7. 失败时写入 `failed` 和错误摘要，物料恢复为可重试状态。

后台任务不得复用请求结束后关闭的 SQLAlchemy Session。

### 4.3 结果页

- 每 2 秒获取一次审查记录。
- `processing`：展示统一的天蓝色加载卡片、阶段说明、已等待时间、返回工作台按钮。
- `completed`：停止轮询并展示现有三栏审查结果。
- `failed`：停止轮询，展示失败原因、重新审查和返回工作台按钮。
- 页面卸载时清理轮询定时器。
- 网络请求失败采用有限次数重试，不把网络错误误判为审查失败。
- 请求达到前端等待阈值后提示“任务仍在后台进行”，允许用户离开；再次进入仍可继续查询。

## 5. 法规数据库设计

### 5.1 分类

页面固定展示五类：

- L1：法律法规。
- L2：行业专项规则，按行业分组。
- L3：行政处罚案例，按现有目录分类。
- L4：平台规则，按平台分组。
- L5：违禁词、改写、免责声明和行业规范模板，按模板目录分组。

不提供搜索框。

### 5.2 正文读取

- 列表接口返回稳定的相对路径标识、标题、分类和分组。
- 正文接口只接受列表接口生成的相对标识。
- 服务端解析路径后必须确认目标仍位于 `KNOWLEDGE_DIR` 内。
- 仅允许读取 `.txt` 普通文件。
- 正文读取失败返回明确的 404 或 400，不返回服务器绝对路径。

### 5.3 页面布局

- 沿用现有天蓝色品牌色、卡片、按钮、圆角、间距和字体层级。
- 顶部使用 L1-L5 分类切换。
- 左侧或上方显示分组，主体显示条目列表。
- 点击条目后显示正文阅读面板，并提供返回列表操作。
- 桌面端和移动端均提供完整的加载、空数据、失败和正文状态。

## 6. 角色与资源权限

### 6.1 市场用户

- 可以提交物料。
- 可以预览上传文件。
- 只能查看、修改和触发审查自己的物料。
- 只能查看自己物料对应的审查记录。

### 6.2 法务用户

- 导航和路由中不显示物料提交入口。
- 直接访问 `/submit` 时跳转至法务工作台。
- 后端拒绝法务用户提交物料或触发 AI 审查。
- 可以查看待法务审核的物料和审查记录。
- 可以提交法务决定，但不能重复覆盖已经完成的决定。

### 6.3 管理员

- 保留所有页面和管理范围内的数据访问权限。

所有规则同时在前端路由/导航和后端依赖/查询层实施，前端隐藏不作为安全边界。

## 7. 安全修复

### 7.1 存储型 XSS

结果页不得把原始广告内容直接作为 HTML 渲染。高亮实现先对原文和匹配词进行 HTML 转义，再插入受控的 `<mark>` 标记；不允许用户输入生成任意标签或事件属性。

### 7.2 上传路径穿越

临时文件不使用客户端文件名作为磁盘路径。服务端生成 UUID 文件名，仅保留经过白名单验证的小写扩展名，并确认文件位于本次临时目录。

### 7.3 文件内容和格式

- 文件预览接口增加登录和市场/管理员角色校验。
- 大小限制从配置读取，并保持服务端强制校验。
- 移除无法可靠解析的旧 `.doc` 格式声明；保留 DOCX。
- 前端预览后的编辑文本作为最终正文，提交时不再与服务端重复提取结果拼接。

### 7.4 资源越权

物料详情、版本记录、审查详情和按物料查询审查均执行资源可见性检查。禁止用户通过猜测 UUID 横向读取他人数据。

### 7.5 AI 异常

- DeepSeek API Key 缺失时返回配置错误并记录失败状态。
- 客户端创建、调用和 JSON 解析均纳入异常处理。
- API 错误不得作为违规规则参与风险扣分。
- 日志记录内部异常，前端只返回清理后的错误摘要。

## 8. 现有流程修复

- 首页物料链接先查询对应 Review，或跳转至正确的物料详情流程，不再把 Material ID 当作 Review ID。
- “提交法务审核”的含义与后端状态一致。AI 完成后自动进入待法务队列，市场端按钮改为状态说明或返回工作台。
- 重新审查复用明确的重试入口，禁止同一物料存在多个并行 `processing` 任务。
- 文件上传正文不重复。
- 法务决定提交后不可重复覆盖。

版本递增和退回重提的完整业务增强保留在后续迭代；本次保证现有版本数据不被破坏。

## 9. 前端一致性约束

- 品牌主色继续使用 `#0EA5E9` 对应的 sky 色阶。
- 优先复用现有 `btn-primary`、`btn-outline`、`card`、`input`、页面标题和布局组件。
- 新增状态页面不得使用孤立的内联视觉体系。
- 所有异步操作必须包含加载、成功、失败、禁用和可恢复操作。
- 市场、法务、管理员的导航项应由同一角色判断来源生成，避免页面与顶栏判断不一致。
- 移动端按钮可点击区域、正文换行和长文件名必须正常显示。

### 9.1 左侧抽屉导航

v0.4.1 采用“左上角菜单按钮 + 左侧可收起抽屉”，替换顶部横向功能选择栏：

- 顶栏左侧显示约 `48 × 48px` 的菜单按钮及 LexAd 品牌名称。
- 抽屉从视口左侧进入，桌面端宽度为 `300px`，移动端宽度不超过 `86vw`。
- 首页、提交物料、法务审核和法规数据库纵向排列。
- 每个菜单项高度约 `56px`，正文字号 `16px`，图标约 `22px`，满足触屏点选需求。
- 当前路由使用天蓝色背景和文字强调。
- 点击菜单项后自动关闭抽屉并执行路由跳转。
- 点击半透明遮罩、抽屉关闭按钮或按下 `Esc` 均可关闭。
- 抽屉打开时锁定页面背景滚动，关闭或组件销毁时恢复。
- 用户名称、角色和退出功能放在抽屉底部，减少顶栏信息密度。
- 菜单项继续使用统一角色来源：法务不显示提交物料，市场不显示法务审核，管理员显示全部功能。
- 使用语义化按钮、导航和可读的 `aria-label`、`aria-expanded`，保持键盘可操作性。

本次仅调整共享布局组件，不改页面路由、后端接口和业务数据模型。
