# LexAd v0.4.0 — 设计文档

**日期**: 2026-05-17
**状态**: 已实施
**范围**: 文件上传与多格式提取、测试物料批量验证、天蓝色系统一与按钮状态修复
**基于**: [v0.3.0 设计](2026-05-15-lexad-v0.3.0-design.md)

## 一、概述

v0.4.0 在 v0.3.0 的"提交→审查→法务决策"核心闭环之上，补齐三项能力：

1. **文件上传与多格式提取** — 从纯文本粘贴升级为文本+文件双通道提交，支持 8 种常见格式，自动提取文本后驱动审查引擎
2. **测试物料批量验证** — 用 65 条标注广告案例端到端验证引擎的合规/违规二分类与风险等级判定
3. **天蓝色系统一** — 品牌色从蓝色(#2563EB)统一为天蓝色(#0EA5E9)，修复按钮 disabled 状态视觉消失

三条线互不阻塞，独立实施，统一汇入 v0.4.0。

## 二、文件上传与多格式提取

> 原单点文件上传设计已整合到本节；更细过程可通过 Git 历史追溯。

### 2.1 新增能力

- 支持格式：JPG / PNG / GIF / BMP / PDF / DOCX / DOC / PPTX / XLSX / TXT
- 文件仅临时处理不持久化，数据库始终只存储文本
- 提取失败自动降级：首选解析 → OCR 兜底 → 提示用户手动输入
- 提交前可预览提取文本并手动编辑

### 2.2 架构

```
前端 SubmitPage（文件拖拽上传 + 预览编辑）
  → POST /api/v1/materials/preview-text（仅提取文本）
  → POST /api/v1/materials/submit（multipart, 可选 file 字段）
    → MaterialService.create_material()
      → FileExtractionService.extract(file)
        → MIME 校验 → dispatch 对应解析器 → OCR 降级
      → 合并提取文本 + 用户输入文本
      → 保存 Material（仅文本）
```

### 2.3 新增模块

| 模块 | 路径 | 说明 |
|------|------|------|
| FileExtractionService | `backend/app/services/file_extraction.py` | 主入口 + 降级编排 + 临时文件管理 |
| ImageParser | `backend/app/services/parsers/image_parser.py` | Tesseract OCR |
| PdfParser | `backend/app/services/parsers/pdf_parser.py` | PyMuPDF 提取 + OCR 降级 |
| OfficeParser | `backend/app/services/parsers/office_parser.py` | DOCX/PPTX/XLSX 文本提取 |
| TextParser | `backend/app/services/parsers/text_parser.py` | TXT 直读 + UTF-8/GBK 编码检测 |

### 2.4 API 变更

- `POST /api/v1/materials/submit`：JSON → multipart/form-data，新增可选 `file` 字段
- `POST /api/v1/materials/preview-text`：新增端点，仅提取文本不保存
- `raw_text` 改为可选，允许纯文件提交

### 2.5 新增依赖

```
PyMuPDF>=1.23.0        # PDF 文本提取
python-docx>=1.1.0     # DOCX 段落提取
python-pptx>=0.6.21    # PPTX 文本框提取
openpyxl>=3.1.0        # XLSX 单元格读取
```

## 三、测试物料批量验证

> 原单点测试验证设计已整合到本节；更细过程可通过 Git 历史追溯。

### 3.1 测试物料

- 文件：`backend/scripts/物料1-65.xlsx`（65 条标注案例）
- T001-T055：违规案例（覆盖 12 个行业）
- T056-T065：合规案例

### 3.2 验证脚本

- 路径：`backend/scripts/validate_materials.py`
- 依赖：`openpyxl` + `requests`
- 执行流程：登录获取 JWT → 逐条提交物料 → 触发 AI 审查 → 获取结果 → 对比期望 → 输出 CSV 报告 → 清理测试数据

### 3.3 对比逻辑

| 条件 | 判定 |
|------|------|
| `ai_risk_score < 80` 或 L1/L2/L3 任一层的 `matched_rules` 非空 | 违规 |
| 以上都不满足 | 合规 |

风险等级：`<50` 高 / `50-79` 中 / `≥80` 低

偏差定义：合规/违规二分类错误 = 严重偏差；风险等级差一档 = 一般偏差；一致 = 通过

### 3.4 成功标准

- 二分类准确率 ≥ 85%
- 风险等级准确率 ≥ 75%

## 四、天蓝色系统一与按钮状态修复

## 五、与 v0.3.0 的关系

v0.4.0 是 v0.3.0 的增量升级：

- **文件上传**扩展了 SubmitPage 的输入能力，不影响审查引擎流水线（引擎仍只消费 `raw_text`）
- **测试验证**为已有引擎提供量化质量度量，不修改引擎代码
- **天蓝主题**是纯视觉层改动，不影响业务逻辑

数据库模型无变更。v0.3.0 的所有 API 端点保持兼容（除 `/submit` 请求格式从 JSON 升级为 multipart 外）。
