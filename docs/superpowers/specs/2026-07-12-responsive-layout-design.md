# LexAd v0.6 响应式布局修复设计

## 背景

当前页面在全屏时可用，但浏览器宽度缩小时存在三类全局问题：登录页在不合适的宽度过早从左右结构切换为上下结构；固定桌面侧栏仍占据视口并遮挡主内容；不同页面各自使用不一致的断点、固定宽度和间距，导致空间关系混乱。登录页的表单控件还存在垂直间距过小、接近重叠的风险。

本设计保持 Vue、UnoCSS 与现有组件结构，不引入大型 UI 框架。目标是建立一套小而统一的流式响应式规则，而不是逐页堆叠补丁。

## 目标与边界

### 目标

- 支持从 `320px` 手机宽度到宽屏桌面的连续缩放。
- 允许控件、栏目和页面内容随可用宽度重排；允许任意高度的纵向滚动。
- 在 `320px` 及以上禁止页面级横向滚动、内容遮挡和控件重叠。
- 将侧栏、顶部导航、主内容偏移和复杂多栏布局的断点统一。
- 保持桌面登录页的左右品牌视觉，手机端才使用上下结构。
- 修复登录表单内按钮、输入框、测试账号区域之间的空间关系。

### 非目标

- 不追求参考 HTML 的像素级复刻。
- 不重写业务逻辑、路由、品牌库或后端接口。
- 不为所有旧页面创建新的布局框架；仅抽取必要的公共容器和规则。
- 不保证小于 `320px` 的视口可用。

## 统一布局模型

| 范围 | 应用壳层 | 登录页 | 内容布局 |
| --- | --- | --- | --- |
| `>= 1280px` | 固定深色侧栏，主内容预留侧栏宽度 | 左品牌区 + 右表单面板 | 允许三栏或双栏 |
| `768px–1279px` | 顶部栏 + 抽屉；固定侧栏与主内容偏移同时移除 | 仍为左右结构，压缩留白、品牌区和表单宽度 | 最多双栏；辅助信息下移 |
| `320px–767px` | 顶部栏 + 抽屉 | 上下结构，精简品牌区 | 单列、自动换行 |

浏览器缩放会改变 CSS 视口宽度，因此电脑窗口缩小到相同宽度时，应进入与手机相同的 CSS 布局状态。它可用于主要视觉回归；最终仍需手机模式检查触摸、虚拟键盘、安全区域和动态地址栏。

## 壳层与全局约束

`AppShell` 是唯一负责桌面侧栏占位的组件。桌面断点以上同时显示固定侧栏和主内容左偏移；低于该断点时，同时隐藏固定侧栏并清除偏移，由移动顶部栏和抽屉接管导航。不得保留只隐藏视觉元素、却继续保留 `margin-left` 的状态。

全局和页面容器遵循下列约束：

- `html`、`body` 与应用根节点禁止页面级 `overflow-x`；不设置会制造横向滚动的文档 `min-width`。
- 所有可收缩的 flex/grid 子项使用 `min-width: 0`；文本在必要时换行或截断。
- 统一页面内边距：桌面、平板、手机递减；卡片内边距同样递减。
- 页面内容使用 `width: 100%`、受控的 `max-width` 和居中容器；不依赖多个页面各自的固定左右留白。
- 页面自然由文档纵向滚动。仅导航抽屉、遮罩、局部列表和确有必要的宽表格可以拥有独立滚动区域。
- 表格横向滚动限定在表格包装容器内，不能扩大 `body` 的滚动宽度。

## 登录页

登录页继续由独立的 `AuthLayout` 承载，绝不渲染登录后侧栏或顶栏。

桌面和紧凑桌面均为左右结构。左侧是渐变品牌背景、标志、标题与能力说明；在紧凑区间逐步减少装饰和留白，而不是直接变成上下结构。表单面板使用流式宽度约束，例如在理想 `480px` 宽度附近通过 `clamp()` 收缩，同时保留可读的最小内容宽度。

仅在小于 `768px` 时切换为上下结构：品牌区域保留标志、标题与副标题，隐藏大装饰和功能列表；表单自然向下延伸。不得依赖固定视口高度或垂直居中造成内容挤压。

输入框与按钮最小触控高度为 `44px`；表单字段、错误消息、提交按钮和测试账号区使用统一的垂直间距令牌，测试账号区不通过负边距或绝对定位接近按钮。

## 页面级重排

- 首页统计卡和快捷入口按三列、两列、单列变化；列表操作允许折行。
- 提交页的上传区、行业选项、平台标签和操作区在窄宽度自动换行或纵向排列。
- 结果页与法务详情页在宽屏使用辅助栏；中等宽度将辅助信息移入主内容流；手机为单列。
- 品牌档案、知识库与管理页面的工具栏允许换行。宽表格采用局部滚动包装。
- 标题行、筛选器与按钮组不得依赖不可收缩的固定宽度；可用空间不足时优先换行而非溢出。

## 验收与测试

在 `320`、`360`、`390`、`768`、`1024`、`1279`、`1280`、`1440px` 宽度，及至少 `568`、`667`、`800`、`900px` 高度进行检查。每个断点检查亮色和深色模式。

关键回归路径为：登录页、登录后首页、导航抽屉、提交页、结果页、法务详情、品牌档案、知识库和管理表格。

每个尺寸必须满足：

1. 页面没有横向滚动条。
2. 固定侧栏不会遮挡内容，且低于桌面断点时不占据主内容空间。
3. 控件、按钮、文本和卡片不重叠；长文本可换行或截断。
4. 所有内容可通过纵向滚动访问。
5. 桌面与手机导航均可操作，登录与登出流程不受影响。
6. `npm run build` 通过；使用浏览器响应式模式进行关键页面人工验收。

## 实施顺序

1. 定义统一断点、页面容器和防横向溢出规则。
2. 重构 AppShell 的侧栏显示与主内容偏移，使其使用同一断点。
3. 调整登录页为三档响应式布局，并修复表单垂直间距。
4. 重排复杂详情页、结果页和表格类页面的栏目与工具栏。
5. 逐尺寸进行浏览器验收，修复仅在特定断点出现的溢出或重叠。
