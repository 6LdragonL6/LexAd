# LexAd v0.6 界面、品牌档案与数据库模式修复实施计划

> 依据：[2026-07-12-v0.6-ui-brand-database-fixes-design.md](../specs/2026-07-12-v0.6-ui-brand-database-fixes-design.md)

## 1. 实施目标

本计划用于修复以下问题，不扩展到其他功能：

1. 登录页恢复参考 HTML 的左右视觉语言，同时保留独立认证布局和手机端适配。
2. 浏览器缩放时，审查详情页的右栏不再挤压或遮挡正文。
3. 登录后桌面端提供明确的退出登录入口。
4. 品牌档案能正确区分加载、空数据、网络失败、权限失败和档案失败。
5. 深色模式建立可见的页面与面板层级。
6. 一键启动默认、明确使用本地 SQLite；Neon 只能显式选择，并且不在普通启动时自动迁移。

本计划不实现 SQLite 与 Neon 同步，也不引入新的 UI 框架或前端测试框架。

## 2. 实施原则

- 开始每个任务前运行 `git status --short`，保留工作区已有修改，在现有差异上增量编辑。
- 前端继续使用 Vue 3、Pinia、Vue Router、UnoCSS 和 `main.css` 令牌。
- 先补语义令牌，再修改组件；避免为每个页面增加一套独立深色 CSS。
- 数据库 URL 只允许通过一个解析函数决定，应用、Alembic 和脚本不得各自实现不同回退规则。
- 本地启动可以自动迁移本地 SQLite；Neon 迁移必须由开发者单独执行。
- 不在日志、浏览器或 PowerShell 输出中打印完整数据库连接串。
- 每个阶段完成后先运行最小相关检查，再继续下一阶段。

### 任务 5.3：建立品牌列表状态机

修改 `frontend/src/pages/BrandArchivePage.vue`。

建议类型：

```ts
type ListState = 'loading' | 'ready' | 'empty' | 'error'
type LoadErrorKind = 'network' | 'forbidden' | 'server' | 'unknown'

interface LoadError {
  kind: LoadErrorKind
  message: string
}
```

用 `listState` 和 `listError` 替代互相可能冲突的 `loading`、`error` 布尔组合。

`loadBrands(query, preserveExisting = false)` 应满足：

1. 首次加载进入 `loading`；搜索重试且已有结果时可以保留当前列表。
2. 请求成功且数组非空：保存结果并进入 `ready`。
3. 请求成功且数组为空：进入 `empty`。
4. 请求失败：进入 `error`；如果要求保留已有结果，不清空品牌数组。
5. `finally` 不再通过单独布尔值覆盖状态。

使用 `axios.isAxiosError` 分类：

- `!error.response`：`network`，文案“无法连接本地后端，请确认一键启动已成功完成。”
- 403：`forbidden`，文案“当前账号无权访问品牌档案。”
- 500 及以上：`server`，优先使用安全的 `detail`，否则“品牌服务暂时不可用。”
- 其他：`unknown`，文案“加载品牌列表失败。”
- 401 不在页面重复处理，继续交给 `client.ts` 的统一认证拦截器。

错误和空状态都使用 `EmptyState`，错误状态通过 action slot 提供“重新加载”按钮。

## 10. 阶段六：显式数据库模式

### 任务 6.1：增加配置字段

修改 `backend/app/core/config.py`：

```python
DATABASE_MODE: Literal["local", "neon"] = "local"
LOCAL_DATABASE_URL: str = "sqlite:///./lexad.db"
DATABASE_URL: str = ""
```

约束：

- `local` 始终使用 `LOCAL_DATABASE_URL`，即使 `.env` 中残留 Neon 的 `DATABASE_URL` 也不能误连云端；
- `neon` 要求 `DATABASE_URL` 非空且为 PostgreSQL URL；
- `APP_ENV=production` 时禁止 `DATABASE_MODE=local`；
- 验证错误不得包含连接串内容。

可以在 Settings 校验器中做静态字段校验，也可以在统一 URL 解析函数中抛出专用 `RuntimeError`。推荐：字段组合约束放 Settings 校验器，URL 选择放解析函数。

### 任务 6.2：新增唯一 URL 解析模块

新增 `backend/app/db/url.py`，只暴露少量清晰接口：

```python
def resolve_database_url(settings: Settings) -> str: ...
def database_target_label(settings: Settings) -> str: ...
def is_sqlite_url(url: str) -> bool: ...
```

行为：

```text
mode=local -> LOCAL_DATABASE_URL
mode=neon + valid DATABASE_URL -> DATABASE_URL
mode=neon + missing/invalid URL -> raise without secret
```

`database_target_label` 只返回 `local SQLite` 或 `Neon PostgreSQL`，绝不返回主机、用户名或密码。

### 任务 6.3：让应用和 Alembic 共用解析逻辑

修改 `backend/app/db/session.py`：

- 调用 `resolve_database_url(settings)`；
- SQLite 才设置 `check_same_thread=False`；
- PostgreSQL 才设置 `pool_size` 和 `max_overflow`；
- 不再以“空 URL 自动回退”作为选择逻辑。

修改 `backend/alembic/env.py`：

- 保留 `.env` 加载；
- 通过 `get_settings()` 和 `resolve_database_url()` 取得 URL；
- 删除独立的 `os.environ.get("DATABASE_URL", ...)` 回退；
- 离线、在线迁移使用同一 URL。

注意 `get_settings()` 有缓存。测试中修改环境变量后必须调用 `get_settings.cache_clear()`。

### 任务 6.4：更新环境变量示例

修改 `backend/.env.example`：

```dotenv
DATABASE_MODE=local
LOCAL_DATABASE_URL=sqlite:///./lexad.db

# 仅 DATABASE_MODE=neon 时读取

```

注释明确：

- 本地默认不连接 Neon；

### 任务 7.2：本地模式启动前迁移和种子

要求：

### 任务 7.3：Neon 模式只做只读预检

`-DatabaseMode neon` 时：

- 要求 `.env` 中有有效 `DATABASE_URL`；
- 执行只读连接/迁移状态检查；
- 不执行 `alembic upgrade head`；
- 若数据库不可达或版本不是 head，终止启动并给出不同错误；
- 输出只允许显示 `Database target: Neon PostgreSQL`。

PowerShell 预检应分别捕获 `python -m alembic current` 和 `python -m alembic heads` 的输出与退出码：

1. `current` 非零退出时报告“Neon 连接或 SSL 会话失败”；
2. `current` 成功后，从两份输出中提取 Alembic revision；
3. current revision 为空或与 heads revision 不同，报告“Neon 迁移版本落后”；
4. 两者相同才允许启动后端；
5. 面向用户的错误只包含分类和 revision，不包含连接串。

当前项目 revision 为 12 位十六进制字符串，可用严格正则提取；不要通过查找输出中是否含有单词 `head` 来判断，因为不同 Alembic 版本的显示格式可能不同。

Neon 人工迁移命令在文档中明确为单独步骤：

```powershell
cd backend
$env:DATABASE_MODE = "neon"
.\venv\Scripts\python.exe -m alembic current
.\venv\Scripts\python.exe -m alembic heads
# 人工确认目标与备份后，才执行：
.\venv\Scripts\python.exe -m alembic upgrade head
```

不要把最后一条命令加入普通一键启动。

脚本手工测试：

2. 重复启动被拒绝。
3. local 数据库迁移失败时不创建服务窗口。
4. stop 后两个端口释放，PID 文件删除。
5. neon 缺少连接串时不创建服务窗口。
6. neon 不可达与迁移落后显示不同提示。

完成标准：双击 `start-dev.bat` 始终使用本地 SQLite，且品牌列表可读取本地 3 个演示品牌。
