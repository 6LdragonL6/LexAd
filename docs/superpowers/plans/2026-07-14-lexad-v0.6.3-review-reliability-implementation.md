# LexAd v0.6.3 审查可靠性实施计划

## 范围

依据 `2026-07-14-lexad-v0.6.3-review-reliability-design.md`，修复业务模型请求、统一真实舆情案例接入，并实现本地证据优先、AI 可信度参与的混合舆情判断。管理员 API 连接测试语义保持不变。

## 阶段 1：DeepSeek 业务请求

1. 为语义审查、舆情案例结构化和舆情风险审查补充明确 JSON 契约及示例。
2. 校验 `finish_reason`，对空响应、截断和结构异常执行最多三次有界尝试。
3. 细分不可重试错误，并记录不包含密钥和完整物料的诊断信息。
4. 增加网关和第二层回归测试。

## 阶段 2：统一案例数据

1. 新增案例规范化模块，同时接受管理员模板和团队真实案例格式。
2. 合并桃李面包重复记录，保留来源、核验信息和别名。
3. 让启动同步、管理员导入和命令行脚本共用规范化逻辑。
4. 停止将 L3 法律案例作为新舆情基线；旧系统生成记录从新快照排除但不删除。
5. 删除空的 `data/case_library.json` 和失效引用。
6. 测试 34 条原始数据、规范化去重、幂等同步和管理员数据保护。

## 阶段 3：舆情融合引擎

1. 触发词只产生候选证据，不直接决定最终中高风险。
2. 使用原始文案、触发摘要、风险维度、标题、品牌、行业和平台执行本地案例召回。
3. AI 在本地无命中或资料库为空时仍独立审查完整物料。
4. 校验 AI 原文证据和案例引用，计算有效可信度。
5. 按本地 60%、AI 40% 的可信度加权融合，处理单侧证据、强本地下限和高可信冲突。
6. 覆盖目标月饼文案、游戏语境“毒打”、AI 不可用、空库和冲突场景。

## 阶段 4：结果、版本与文档

1. 结果页展示最终分数、证据来源、原文证据和人工复核冲突提示，并兼容旧结果。
2. 更新前后端版本为 0.6.3。
3. 新增发布说明并更新 README、文档索引和技术参考。

## 阶段 5：验证

1. 运行 DeepSeek 网关、案例同步、舆情融合和审核任务针对性测试。
2. 运行全部后端测试。
3. 运行前端类型检查与生产构建。
4. 检查工作区差异、敏感信息和版本一致性。
