# LexAd v0.6.2 管理能力与登录稳定性实施计划

来源设计：[LexAd v0.6.2 管理能力与登录稳定性设计](../specs/2026-07-14-lexad-v0.6.2-design.md)。

## 实施约束

- 只提供单一 DeepSeek API Key 配置；Base URL 固定为 `https://api.deepseek.com`，模型固定为 `deepseek-v4-flash`。
- 真实 API Key 不进入 Git、日志、审计记录、接口响应、浏览器持久化或自动化测试。
- 回收站覆盖物料及审核关系、品牌、舆情案例和平台规则，保留期固定为 15 天。
- 导入记录和审计日志永久保留。
- 采用增量迁移并兼容 SQLite 与 PostgreSQL，不重建现有审核状态机。
- 每阶段先补充或运行相应测试，保留用户已有的无关改动。

## 阶段 1：数据库模型与迁移

主要文件：

- `backend/app/models/admin.py`（新增）
- `backend/app/models/material.py`
- `backend/app/models/brand.py`
- `backend/app/models/knowledge.py`
- `backend/app/models/__init__.py`
- `backend/alembic/versions/*_v0_6_2_admin_controls.py`（新增）

工作项：

1. 新增单例安全配置模型，保存加密值、尾号指纹、验证状态和更新人。
2. 新增统一回收站索引，保存对象类型、对象 ID、摘要、删除/清理时间和操作人。
3. 为物料、品牌、舆情事件和平台规则集增加 `deleted_at`、`deleted_by_id`、`purge_after` 字段及索引。
4. 复用或扩展现有知识审计日志记录管理操作，避免记录请求体和完整物料。
5. 编写兼容 SQLite/PostgreSQL 的 Alembic 升降级迁移。

验收：临时数据库迁移后现有数据仍可见；新增表和字段可被 Alembic 正确发现。

## 阶段 2：安全配置与模型网关

主要文件：

- `backend/app/services/admin_settings_service.py`（新增）
- `backend/app/services/deepseek_gateway.py`（新增）
- `backend/app/services/model_service.py`
- `backend/app/engine/layer2_semantic.py`
- `backend/app/engine/public_opinion.py`
- `backend/app/core/config.py`
- `backend/app/tests/test_admin_settings.py`（新增）
- `backend/app/tests/test_deepseek_gateway.py`（新增）

工作项：

1. 使用由 `SECRET_KEY` 派生的 Fernet 密钥加密 API Key；默认生产密钥禁止保存。
2. 实现只返回状态、尾号和更新时间的配置读取、保存、测试与清除服务。
3. 保存新 Key 前使用固定模型完成最小 JSON 请求，失败时保留旧 Key。
4. 建立统一网关，固定 Base URL/模型、请求超时、两次有界重试、JSON 输出模式和错误分类。
5. 为三类任务定义 Pydantic 输入/输出契约，统一格式化、截断和验证。
6. 移除业务模块中的直接 OpenAI 客户端创建和直接环境变量 Key 读取。
7. 模型失败时保留确定性审查结果，并将模块标记为失败或不可用。

验收：生产代码只有网关创建 OpenAI 客户端；测试全部使用伪造 Key 和 Mock，不产生外部调用。

## 阶段 3：回收站领域服务与清理任务

主要文件：

- `backend/app/services/recycle_bin_service.py`（新增）
- `backend/app/services/material_service.py`
- `backend/app/services/brand_service.py`
- `backend/app/services/admin_knowledge_service.py`
- `backend/app/services/review_service.py`
- `backend/app/main.py`
- `backend/app/tests/test_recycle_bin.py`（新增）

工作项：

1. 实现四类根对象的删除、恢复、永久清理和统一分页列表。
2. 物料删除时中止执行中的审核；恢复正在审查的物料时转回草稿。
3. 品牌删除不删除历史物料；永久清理前解除物料品牌外键。
4. 舆情事件与版本、平台规则集与版本按关系图整体恢复和清理。
5. 所有普通列表、详情、统计、队列与审查选择统一排除软删除对象。
6. 应用启动时清理一次，运行期间每 24 小时执行；单对象失败不阻断其他对象。
7. 所有操作写入永久审计日志，自动任务使用系统主体。

验收：四类对象在 15 天内可恢复，到期可幂等清理，且没有孤儿记录。

## 阶段 4：管理员后端接口

主要文件：

- `backend/app/api/v1/endpoints/admin_settings.py`（新增）
- `backend/app/api/v1/router.py`
- `backend/app/schemas/admin.py`（新增）
- `backend/app/tests/test_admin_settings_api.py`（新增）

工作项：

1. 提供配置状态、保存并验证、测试当前配置和清除接口。
2. 提供回收站列表、删除业务对象、恢复和永久清理接口。
3. 所有接口使用 `require_admin`；普通角色返回 `403`。
4. API 响应不包含密钥明文、加密值、完整物料或上游敏感错误。

验收：管理员可以完成所有管理动作，非管理员无法通过直接请求绕过权限。

## 阶段 6：登录稳定性修复

主要文件：

- `backend/app/schemas/auth.py`
- `backend/app/api/v1/endpoints/auth.py`
- `backend/app/services/auth.py`
- `frontend/src/api/client.ts`
- `frontend/src/api/auth.ts`
- `frontend/src/stores/user.ts`
- `frontend/src/pages/LoginPage.vue`
- 认证相关测试

工作项：

1. 登录成功响应直接包含用户数据，移除前端登录后的第二次 `/me` 请求。
2. 修正响应拦截器：登录 `401` 只返回表单；仅携带会话令牌的非登录请求触发过期处理。
3. 登录状态原子更新，失败时不留下令牌或用户。
4. 增加重复提交保护、请求超时、可区分的安全错误提示和可等待的路由替换。
5. 后端对不存在用户、错误密码和异常哈希使用一致安全失败路径。

验收：连续错误登录、快速点击、断网和超时均不刷新整页、不锁死按钮、不产生未处理异常。
