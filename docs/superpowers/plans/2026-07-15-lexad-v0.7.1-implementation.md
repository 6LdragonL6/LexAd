# LexAd v0.7.1 实施计划

> 依据：`docs/superpowers/specs/2026-07-15-lexad-v0.7.1-review-version-brand-industry-design.md`

## 目标

发布 LexAd v0.7.1，修复重提后的标题与内容串版、品牌关联丢失和法务详情缺少舆情信息的问题；把品牌行业从单值自由文本升级为标准多行业与管理员候选确认机制；重构法务审核台为中栏主内容布局，并保持 README 的主要展示思路不变。

## 实施原则

- 先用回归测试固定现有故障，再修改模型和接口。
- Review 绑定的提交快照是审核执行和详情展示的唯一版本化数据源。
- 工作台稳定标题、最新工作副本与历史提交快照职责分离。
- 品牌在首次提交后锁定；行业按每次提交保存，并通过候选机制沉淀到品牌档案。
- 前后端同时执行状态、权限与品牌约束，不能只依赖界面禁用。
- 新增视觉元素复用主题变量，并分别验证明暗模式。
- 每一阶段完成后先运行定向测试，最终再运行全量测试和生产构建。
