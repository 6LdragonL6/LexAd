# LexAd v0.6 发布收尾实施计划

来源设计：[v0.6 发布收尾设计](../specs/2026-07-12-lexad-v0.6-release-polish-design.md)。

## 实施约束

- 在用户已有未提交修改上增量工作，不回退无关文件。
- 不引入大型 UI 框架，不改变品牌库核心业务范围。
- 每个阶段完成后运行与风险相称的构建或测试。
- Neon 的迁移和写入必须独立确认；日常启动不得自动修改云数据库。

## 阶段 1：主题与公共布局基础

主要文件：

- `frontend/src/styles/main.css`
- `frontend/uno.config.ts`

工作项：

1. 补齐页面、外壳、普通卡片、抬升面板、内嵌区域、悬停、边框和文字语义令牌。
2. 将深色 UnoCSS 兼容映射分别对应不同表面层级。
3. 定义统一页面容器、响应式工具栏、卡片内边距和表格局部滚动规则。
4. 全局防止页面级横向溢出，但不设置文档最小宽度。
5. 确保可收缩的 flex/grid 子项使用 `min-width: 0`。

验收：亮色与深色层级清晰，320px 页面不产生 body 横向滚动，前端构建通过。

## 阶段 2：应用壳层与导航

主要文件：

- `frontend/src/layouts/AppShell.vue`
- `frontend/src/components/layout/AppSidebar.vue`
- `frontend/src/components/layout/MobileNavigation.vue`
- `frontend/src/components/layout/TopBar.vue`
- `frontend/src/styles/main.css`

工作项：

1. 固定侧栏和主内容偏移统一在 1280px 切换。
2. 低于 1280px 显示移动顶栏和抽屉，并清除桌面偏移。
3. 保持抽屉触控尺寸、Escape 关闭、遮罩关闭和背景滚动锁定。
4. 桌面顶栏与移动抽屉都接入统一登出流程。
5. 手机宽度隐藏非必要口号，但保留品牌和菜单入口。

验收：1279px 无固定侧栏，1280px 主内容精确避开侧栏；登录后可从两种导航退出。

## 阶段 3：登录页

主要文件：

- `frontend/src/layouts/AuthLayout.vue`
- `frontend/src/pages/LoginPage.vue`
- `frontend/src/styles/main.css`

工作项：

1. 保持登录页只渲染 `router-view`，不进入 AppShell。
2. 宽屏保留完整品牌装饰和约 480px 表单面板。
3. 768–1279px 保持左右布局，压缩品牌留白并隐藏大装饰。
4. 低于 768px 切换为上下布局，隐藏能力列表。
5. 修复标题、字段、错误、按钮和测试账号区的垂直间距。
6. 矮窗口使用页面纵向滚动，控件最小高度 44px。

验收：320px 与 768px 无横向溢出；568px 高度下按钮和测试账号区不重叠。

## 阶段 4：复杂页面响应式迁移

主要文件：

- `frontend/src/components/layout/ThreeColLayout.vue`
- `frontend/src/pages/HomePage.vue`
- `frontend/src/pages/SubmitPage.vue`
- `frontend/src/pages/ResultPage.vue`
- `frontend/src/pages/LegalDashboard.vue`
- `frontend/src/pages/LegalDetail.vue`
- `frontend/src/pages/BrandArchivePage.vue`
- `frontend/src/pages/KnowledgePage.vue`
- `frontend/src/pages/AdminKnowledgePage.vue`

工作项：

1. 页面外层统一迁移到流式容器，保留响应式上下与左右留白。
2. 标题、筛选器和按钮组允许换行；长文本可换行或截断。
3. 首页卡片按三列、两列、单列重排。
4. 提交页上传区、行业、平台和更多设置在窄屏重排。
5. 三栏法务详情只在 1280px 以上启用独立栏；其余宽度回到自然文档流。
6. 结果页和品牌档案双栏只在宽屏启用。
7. 知识库标签和管理表格使用局部横向滚动。

验收：关键页面在 320、768、1024、1279、1280 和 1440px 无页面级横向滚动或遮挡。

## 阶段 5：品牌档案与 API 约束

主要文件：

- `backend/app/api/v1/endpoints/brands.py`
- `backend/app/schemas/brand.py`
- `frontend/src/pages/BrandArchivePage.vue`
- `frontend/src/components/common/EmptyState.vue`

工作项：

1. 品牌端点使用具体 Pydantic 响应模型。
2. 前端实现列表和档案两个独立状态机。
3. 区分网络、权限和服务端错误，提供重试入口。
4. 搜索或档案失败时保留仍然有效的数据。
5. 归档后从 active 列表移除品牌并清理选择。
6. 添加品牌端点返回结构和错误路径测试。

验收：成功、空数组、无响应、401、403、500 和档案失败均表现正确。

## 阶段 6：数据库模式

主要文件：

- `backend/app/core/config.py`
- `backend/app/db/url.py`
- `backend/app/db/session.py`
- `backend/alembic/env.py`
- `backend/.env.example`
- `backend/app/tests/test_database_config.py`

工作项：

1. 新增显式 `DATABASE_MODE=local|neon`。
2. local 模式拒绝非 SQLite URL；neon 模式拒绝缺失或非 PostgreSQL URL。
3. 应用与 Alembic 共用同一解析函数。
4. 生产环境拒绝 local 模式。
5. 测试默认值、合法/非法组合、环境变量优先级和 Alembic 一致性。

验收：配置测试通过，local 模式没有连接云数据库的路径。

## 阶段 7：启动脚本

主要文件：

- `scripts/start-dev.ps1`
- `start-dev.bat`
- `stop-dev.bat`
- `docs/local-development.md`

工作项：

1. 确保 `param(...)` 是 PowerShell 脚本第一条语句，并兼容 Windows PowerShell 5.1 编码。
2. local 模式先迁移和种子，再启动服务。
3. neon 模式只读检查连接与 revision，缺少配置时立即停止。
4. 子进程命令正确延迟展开 `DATABASE_MODE`。
5. 使用稳定正则提取 Alembic revision。
6. 后端健康检查成功后再启动或确认前端。
7. 关闭脚本只停止本次启动记录的进程。

验收：PowerShell 静态解析通过；完整执行一键启动、登录、品牌列表和一键关闭。

## 最终验证

1. `cd frontend; npm run build`
2. `cd backend; python -m pytest app/tests -q`
3. 空临时 SQLite 执行 `alembic upgrade head`。
4. 种子脚本连续运行两次，用户和品牌不重复。
5. 浏览器检查 320、360、390、768、1024、1279、1280、1440px。
6. 检查亮色、深色、登录、登出、抽屉、品牌档案和法务详情。
