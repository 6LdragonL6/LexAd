# LexAd v0.7.1 补充修复实施计划

日期：2026-07-15

依据：[品牌创建、启动稳定性与品牌记忆补充设计](../specs/2026-07-15-lexad-v0.7.1-brand-creation-startup-memory-followup-design.md)

## 目标

在不新增数据库迁移、不改变品牌创建 API 契约、不改变 README 主叙事的前提下，完成品牌创建失败可恢复、Windows 一键启动稳定化，以及品牌档案完整记忆印象。

## 任务 1：后端品牌记忆契约

涉及文件：

- `backend/app/schemas/brand.py`
- `backend/app/services/brand_service.py`
- `backend/app/tests/test_v071_brand_memory.py`

步骤：

1. 先增加覆盖空样本、少量样本、稳定、波动、重点关注、建议去重和稳定排序的失败测试；
2. 新增明确的品牌记忆 schema，包括状态、摘要、样本、近期决定、修改倾向、风险、建议和行业；
3. 从现有物料、已完成审核、法务决定与审核建议确定性归纳记忆；
4. 将 `memory_impression` 接入 `BrandProfile`，保持默认值与旧调用兼容；
5. 运行品牌服务相关测试与完整后端回归。

## 任务 2：品牌创建受控异步交互

涉及文件：

- `frontend/src/components/brand/BrandSelector.vue`
- `frontend/src/pages/SubmitPage.vue`

步骤：

1. 将创建中和创建错误状态提升到提交页；
2. 选择器触发创建时不再立即清空或关闭；
3. 成功更新 `modelValue` 后再清空，失败时保留原文字和可见错误；
4. 区分搜索错误和创建错误，并禁止重复创建；
5. 确认同名品牌 `created=false` 路径仍会选中品牌；
6. 运行 TypeScript 检查。

## 任务 3：共享品牌记忆展示

涉及文件：

- `frontend/src/types/index.ts`
- `frontend/src/components/brand/BrandMemoryCard.vue`
- `frontend/src/components/brand/BrandProfilePanel.vue`
- `frontend/src/pages/BrandArchivePage.vue`
- 现有紧凑模式调用页面

步骤：

1. 增加品牌记忆 TypeScript 类型；
2. 为 `BrandMemoryCard` 增加 `compact/full` 模式；
3. 紧凑模式保持审核侧栏体量，完整模式展示摘要、样本、近期决定、修改轮次、风险、建议和行业；
4. 在品牌档案右侧顶部放置完整记忆卡；
5. 从详细面板移除被完整记忆卡吸收的重复指标和高频违规区块；
6. 检查明暗模式状态色、背景、边框和空状态。

## 任务 4：Windows 一键启动稳定性

涉及文件：

- `scripts/start-dev.ps1`
- `docs/guides/local-development.md`

步骤：

1. 新增 `-BackendReload` 开关，默认后端命令不含 `--reload`；
2. 为每次启动生成标识和后端退出状态文件，启动前清理旧标记；
3. 后端窗口命令在 Uvicorn 退出后写入退出码、时间和启动标识；
4. 健康检查改为 `127.0.0.1`，等待 60 秒并保存最后错误；
5. 发现当前后端退出标记时提前结束并输出区别化诊断；
6. 保持 PID 文件、数据库预检与停止脚本行为不变；
7. 使用 PowerShell parser 做语法检查，并静态验证默认/重载命令分支。

## 任务 5：发布收尾

涉及文件：

- `README.md`（仅必要质量状态）
- `docs/releases/v0.7.1-release-notes.md`
- `docs/guides/user-guide.md`
- `docs/guides/local-development.md`

验证：

1. 后端完整回归；
2. 前端 `vue-tsc --noEmit`；
3. PowerShell 语法解析；
4. 全新 SQLite 迁移到 Alembic head；
5. Markdown 链接检查；
6. `git diff --check`；
7. 尝试 `npm run build`，若环境仍限制 esbuild，准确记录结果。

## 完成标准

- 创建品牌失败后名称和错误保持可见，可直接重试；
- 默认一键启动不启用后端重载，并能区分提前退出与健康超时；
- 品牌档案展示与审核界面同源的完整记忆印象；
- 无数据库迁移；
- 全量后端测试和前端类型检查通过；
- v0.7.1 发布说明包含本轮补充修复。
